## 2. 의사결정 프레임워크 (2-Step Max)

> **"차라리 생각을 좀 하고 그 다음에 액션을 취하고"** — 폴 김 목사

### 유형 A: 루틴 결정 (에이전트 자율 실행)

에이전트가 스스로 판단하고 실행한 후 보고한다.

```
해당 조건:
  ✅ 기존 가이드라인 안에 있는 작업
  ✅ 리스크 낮음 (데이터 수정, 콘텐츠 업데이트)
  ✅ 예산 $0 (무료 도구·API 사용 범위)

흐름:
  에이전트 실행 → 실행 → ✅ 완료 보고 (hebronplatform@gmail.com)
  
예시:
  - 주간 SNS 콘텐츠 생성
  - 도시 데이터 오류 수정
  - 주간 재무 보고서 생성
  - 깨진 링크 감지·보고
```

### 유형 B: 전략적 결정 (에이전트 제안 → CEO 승인)

```
해당 조건:
  🔸 새 파트너십·계약 관련
  🔸 예산 집행 ($50 이상)
  🔸 새 도시 런치·삭제
  🔸 브랜드 방향 변경
  🔸 대외 발표 (보도자료·공식 SNS)

흐름:
  에이전트 제안서 작성
        ↓
  CEO(폴 김 목사) 검토 → "예" 또는 수정
        ↓
  에이전트 실행 → ✅ 완료 보고

목표 응답 시간: 24시간 이내
```

### 유형 C: 긴급 상황 (즉시 실행 + 즉시 통보)

```
해당 조건:
  🚨 사이트 다운
  🚨 보안 위협 감지
  🚨 심각한 데이터 오류 (잘못된 정보 노출)
  🚨 언론 부정 보도

흐름:
  에이전트 즉시 조치 → 🚨 SMS·이메일 즉시 통보
  
원칙: "먼저 막고, 나중에 보고"
```

### 의사결정 7질문 (모든 결정 전 자동 통과)

```
∞. 이것이 하나님의 영광을 위한 것인가? (고전 10:31)
🌿 섬기러 왔는가? (막 10:45)
★ 황금률을 적용했는가? (마 7:12)
0. 내가 사용자라면 이걸 원할까?
1. 시장·문화를 정확히 이해한 결정인가?
2. 효과적으로 목적을 달성하는가?
3. 사람을 수단으로 쓰지 않는가?

→ ∞번 NO이면 시작도 하지 않는다
```

## 3. 에이전트 주간 운영 리듬 (Weekly Cadence)

```
월요일 (Monday)     ─── CMO (Marketing Agent)
  → 주간 카카오톡 카드 + SNS 콘텐츠 플랜
  → 인스타그램·페이스북·X 예약 게시
  → 이번 주 성과 목표 설정

화요일 (Tuesday)    ─── CCO (Content Agent)
  → 도시 행사 업데이트 (이번 주 행사 확인)
  → 데이터 리프레시 (폐업·이전 감지)
  → 새 도시 리서치 1-2개 착수

수요일 (Wednesday)  ─── CPO (Partnership Agent)
  → 교회 아웃리치 이메일 발송 (5개 교회/주)
  → 파트너 팔로업 (답장 체크·후속 연락)
  → World Relief·기업 파트너십 진행 상황 업데이트

목요일 (Thursday)   ─── CFO (Finance Agent)
  → 주간 수익·비용 리포트 자동 생성
  → 이상 지출·수익 감소 경고 플래그
  → 월간·연간 목표 대비 진행률 계산

금요일 (Friday)     ─── CTO (Claude Code / Dev Lead)
  → 기술 개선 작업 (버그 수정·기능 추가)
  → 새 도시 코드 추가·배포
  → 성능 최적화·SEO 개선

토요일 (Saturday)   ─── CPR (PR Agent)
  → 보도자료 초안 또는 미디어 피치 작성
  → 한인 미디어 (미주중앙일보·한국일보 등) 제출
  → 다음 주 미디어 기회 스카우팅

일요일 (Sunday)     ─── CEO 검토 시간
  → 전 에이전트 주간 완료 보고 확인
  → 다음 주 우선순위 승인
  → 영적 점검 — "이 방향이 하나님 뜻인가?"
  → (배포·상업 SNS는 일요일 금지 — Chick-fil-A 원칙)
```

## 5. 에이전트 협업 구조 (Collaboration Matrix)

### 의존 관계

```
Marketing Agent (CMO)
  ← Content Agent 에서: 도시 데이터·행사 정보 (콘텐츠 재료)
  ← Finance Agent 에서: 예산 승인 (유료 광고 집행 시)
  → PR Agent 에게: 성공 캠페인 사례 (보도자료 재료)

Partnership Agent (CPO)
  ← PR Agent 에서: 미디어 보도 자료 (파트너에게 신뢰 자료)
  ← Content Agent 에서: 도시별 콘텐츠 현황 (파트너 제안서 재료)
  → Finance Agent 에게: 파트너십 수익 데이터

Content Agent (CCO)
  ← Dev Lead 에게: 코드 구조 이해 (어디에 데이터 넣는지)
  → Marketing Agent 에게: 콘텐츠 재료 공급
  → Dev Lead 에게: 새 도시 데이터 (코딩용)

Finance Agent (CFO)
  ← 모든 에이전트에서: 수익 창출 활동 데이터
  → CEO 에게: 주간·월간 재무 보고
  → 모든 에이전트에게: 예산 한도 안내

PR Agent (CPR)
  ← Marketing Agent 에서: 성공 캠페인·숫자
  ← Partnership Agent 에서: 파트너십 뉴스
  → Partnership Agent 에게: 미디어 보도 자료

Dev Lead (CTO)
  ← Content Agent 에서: 도시 데이터 (코딩용)
  → Content Agent 에게: 코드 배포 완료 확인
  → 모든 에이전트에게: 기술 지원

UX Agent (CXO)
  ← 모든 채널에서: 사용자 피드백·불만
  → Dev Lead 에게: UX 개선 요청
  → Marketing Agent 에게: 사용자 인사이트
```

### 주간 협업 흐름

```
월 (Marketing 계획)
 → 화 (Content가 재료 공급)
  → 수 (Partnership이 교회에 콘텐츠 공유)
   → 목 (Finance가 주간 수익 집계)
    → 금 (Dev가 기술 개선)
     → 토 (PR이 성과 미디어화)
      → 일 (CEO 전체 검토)
```

## 8. AI 안전장치 및 한계

### 반드시 사람(폴 김 목사)이 결정할 영역 (AI 자율화 ❌)

```
1. 새 파트너 교회 최종 확정 — 영적 분별이 필요
2. 법인·금융 계약 서명 — LLC·세금·법적 구속
3. 이단 판별 — 교회 등재 거부 결정
4. 위기 대응 — 언론 부정 보도·소송·사역 갈등
5. 전략 방향 전환 — 새 앱 출시·사업 모델 변경
6. 직원·파트너 해고·관계 종료
```

### AI 자율성 단계

```
현재 (Level 1):
  AI 제안 → 사람 100% 검수 → 실행

3개월 후 (Level 2):
  AI 자동 실행 → 사람 무작위 샘플 점검 (20%)

1년 후 (Level 3):
  AI 완전 자율 → 사람 KPI만 모니터
```

### 감사 추적 (Audit Trail)

모든 AI 작업은 기록:
- 날짜·시간
- AI 모델 (Claude Sonnet 4.6)
- 입력·출력 요약
- 사람 승인 여부
- 결과 (성공/실패)
