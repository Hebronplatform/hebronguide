# 환대 이야기 → eBook 전략 (Story to eBook)

> HebronGuide 3기둥 중 **① 문화 상품 제작**의 첫 실물.
> 흩어진 환대 경험담을 모아·엮어 **따뜻한 이야기 선집(eBook)**으로 세상을 밝힌다.
> — 폴 김 목사 발의 (2026-07-13)

---

## 0. 한 문장 정의

> **"낯선 도시에서 받은·베푼 환대의 짧은 이야기들을, 검증·동의를 거쳐 테마별 선집으로 엮어 무료로 나누는 디아스포라 환대 이야기집."**

Chicken Soup for the Soul은 **상표·형식**이므로 모방·차용하지 않는다. 벤치마킹은 하되 **우리 고유 정체성**으로: *디아스포라(흩어진 한인) · 환대 · 그리스도의 빛*. 가제 후보 — 「낯선 도시, 따뜻한 손」 / 「나그네의 밥상」 / *Strangers Welcomed*.

---

## 1. 왜 (사명 정렬)

| 기둥/원칙 | 연결 |
|---|---|
| 3기둥 ① 문화 상품 제작 | eBook = 첫 문화 상품 실물 |
| "보여주는 전도"(간접 증거) | 강요 없는 환대 이야기가 그리스도의 빛을 자연히 드러냄(마 25:35) |
| 언러닝 전략 | "AI가 못 하는 것 = 환대·맥락·인과관계" — 실제 사람의 이야기가 그 해자 |
| 자생적 KPI 사이클 | 보여주는 성도 → 호감 접근 → 첫 관문 → 확산. eBook이 "보여주기"를 증폭 |

**주의(선교 신학 함정):** 이야기를 전도 도구로 **소비**하지 않는다. 사람을 수단화 금지(Hard Rule). 이야기의 주인공은 하나님과 그 사람이지 HebronGuide가 아니다.

---

## 2. 지금 있는 것 (재사용 자산 — 새로 안 만듦)

```
수집  story-submit.html (2층 폼)         ── 있음
게이트 /api/submit-story (AI Haiku 검증)   ── 있음
저장  Supabase community_items           ── 있음 (type=story_share / story_studio)
검토  admin.html '이야기' 탭              ── 있음 (confidence·Approve/Reject)
발행  story-{city}-01.html 등 웹 발행글    ── 있음 (샘플)
진입  랜딩 nav·히어로 CTA·앱 홈 배너       ── 있음
```

→ **eBook에 새로 필요한 것은 딱 3가지**: ① 저자 **동의·표기** 필드, ② **선집 큐레이션** 워크플로(목차·테마 태그), ③ **조판·내보내기**(웹 선집 + PDF/EPUB).

---

## 3. 무엇을 · 어디서 · 어떻게 (파이프라인 5단계)

```
① 수집(Collect)  →  ② 검증·동의(Verify & Consent)  →  ③ 큐레이션·편집(Curate)
     →  ④ 발행(Publish: 웹 먼저 → 선집)  →  ⑤ 배포·확산(Distribute)
```

### ① 수집 — "주제 캠페인"으로 질을 끌어올림
막연한 "이야기 주세요"보다 **한 번에 하나의 주제**로 초청(응답률·품질↑, 목차도 자동 생성).
- 예시 주제: "처음 도착한 그날 밤, 누군가의 환대" / "낯선 이의 밥상" / "말이 안 통했지만 통한 순간" / "교회에서 받은 첫 환영" / "내가 누군가에게 베푼 환대".
- 채널: story-submit(있음) + 도시별 오픈채팅 + 협력교회 주보 + 한인회.

### ② 검증·동의 — **eBook의 핵심 관문 (Hard Rule)**
AI 게이트(있음)로 세계관·이단·스팸·환대톤 1차 필터. **그 위에 사람 검토 + 저자 동의**를 반드시 얹는다:
- **수록 동의**: "이 이야기를 HebronGuide 무료 선집(웹·eBook)에 실어도 좋습니다" 체크.
- **표기 방식**: 실명 / 이니셜 / 익명 / 필명 중 선택.
- **연락 동의**: 편집·확인 위해 연락 가능(선택).
- **사실 여부**: AI는 편집만 돕고 **사실을 지어내지 않는다**. 미검증·미동의 이야기는 발행 금지.

### ③ 큐레이션·편집 — 사람이, 목회적 톤으로
- admin '이야기' 탭에 **"선집 후보" 태그 + 테마 분류 + 순서(목차)** 추가.
- 편집 가이드: 짧게(1~2쪽), 환대·따뜻함, 정치·교리 논쟁 배제, 성경적 세계관 필터, 개인정보 보호.
- 각 편 구성: **제목 · 짧은 이야기 · 성찰 한 줄 · 도시 태그**.

### ④ 발행 — 웹 먼저, 선집은 그다음
- **웹 우선**: 검증·동의된 이야기는 story 발행 페이지·앱 피드에 먼저(SEO·즉시 확산).
- **선집(eBook)**: 웹에 쌓인 것 중 골라 분기/연간 조판. 포맷 = **웹 선집 페이지(무료 읽기) + PDF(다운로드) + EPUB(선택)**. 표지·서문(목사님)·챕터 도입·부록(참여법·HebronGuide 소개).

### ⑤ 배포·확산 — 기본 무료
- 무료 다운로드(지역 투자·영구 무료 원칙) + 카톡·링크 공유 + 공유 추적→대시보드.
- 협력교회·한인회 배포. 원하면 **후원/기부**로 지속(강요 없음).

## 4. 책의 구도 (예시 목차)

```
표지 · 서문(목사님) — "나그네 되었을 때 너희가 영접하였다"(마 25:35)
1부. 도착 — 처음 그 도시에 선 날
2부. 환대 — 낯선 이가 건넨 손·밥·잠자리
3부. 언어 너머 — 말이 안 통해도 통한 순간
4부. 교회 — 첫 환영, 공동체가 된 사람들
5부. 회복 — 무너졌다가 다시 일어선 이야기
6부. 베풂 — 이제 내가 누군가의 환대가 되다
부록 — 당신의 이야기 참여법 · HebronGuide 소개
```
각 부: 짧은 이야기 6~10편 + 부 도입 한 단락. 도시 태그로 지역 다양성 확보.

## 5. 원칙·법·윤리 (반드시)

| 항목 | 규칙 |
|---|---|
| 저작권 | 저자가 저작권 보유. HebronGuide는 **수록·배포 라이선스**만(동의서). |
| 상표 | "Chicken Soup" 명칭·형식 차용 금지 → **자체 브랜드**. |
| 개인정보 | 실명/익명 선택. 민감정보(주소·연락처) 비공개. 미성년자 보호자 동의. |
| 재정 | LLC(수익)·교회(사역) **분리**. 기본 무료. 데이터 판매 금지. |
| 안식 | 일요일 상업·배포 활동 없음. |
| 진정성 | AI는 편집 보조만. 사실 날조·미검증 발행 금지(Hard Rule). |

---

## 6. 목표 (측정 가능 — 파일럿)

```
파일럿(1권):  3개월 25~30편 검증·동의 확보  →  1부작 선집 발행(웹+PDF)
연간화:       도시별 최소 3편 · 연 1권(50~70편) · 한/영 병기
성공 지표:    참여 편수 · 발행 편수 · 다운로드/공유 수 · (자생) 검색 유입
```
다운로드·조회 '숫자'가 목적이 아니라 **환대 사이클이 도는가**가 진짜 KPI(언러닝 원칙).

---

## 7. 무엇부터 (가장 작은 첫걸음 — Phase 0)

**"생각 먼저, 계획 확인 후 실행" 원칙 — 코드는 최소로, 검증부터.**

1. **동의·표기 필드 추가** (story-submit + submit-story): "eBook 수록 동의" 체크 + 표기 방식(실명/익명/필명). → 지금부터 들어오는 이야기가 곧바로 선집 후보가 됨. *(작은 코드 변경)*
2. **첫 주제 캠페인 1개 오픈**: "처음 도착한 그날 밤, 누군가의 환대". story-submit 히어로에 주제 배너. *(콘텐츠)*
3. **파일럿 목표 확정**: 3개월 25편, 첫 선집 1부작. *(결정)*

Phase 1 = 수집 가속 · Phase 2 = admin 선집 큐레이션 워크플로 · Phase 3 = 첫 PDF 조판 · Phase 4 = 배포·후원·연간화.

## 9. 실행 순서 (확정 반영)

```
Phase 0 (지금):  ① 저자 '수록 동의 + 표기(실명/익명/필명)' 필드 (story-submit + submit-story)
                 ② 첫 주제 캠페인 배너 "처음 도착한 그날 밤의 환대"
                 → 오늘부터 들어오는 이야기 = 동의된 선집 후보
Phase 1 (수집):  주제 캠페인 3개월 운영 · 협력교회/한인회/오픈채팅 초청 · 목표 25편
Phase 2 (큐레이션): admin '이야기' 탭에 '선집 후보' 태그 + 테마 분류 + 목차 순서
Phase 3 (조판):  검증·동의 25편 → 6부 목차로 웹 선집 + PDF 조판 (서문·표지·부록)
Phase 4 (배포):  무료 다운로드 + 카톡·링크 공유 + 후원 안내 → 연간화
```
