**핵심: 새 인사이트 추가 시 자동 정리·중복 제거·시각 품질 유지**

## 🎯 자동 위생 5단계 (Claude Code 자동 실행)

새 인사이트·전략·콘텐츠 추가 시마다 자동 실행:

```
[1단계: 추가] 새 내용을 적절한 문서에 추가
[2단계: 감지] 기존 문서와 중복·유사 내용 검색
[3단계: 통합] 중복 발견 시:
   - 정본 문서 1곳만 권위 있게 유지
   - 다른 곳은 링크로 참조
[4단계: 정렬] 시각 품질 점검 (테이블·아이콘·계층)
[5단계: 동기화] README·CLAUDE.md 인덱스 갱신
```

3. 변경 이력에 기록

## 🎨 시각 품질 표준

### 좋은 문서의 특징

| 특징 | 적용 |
|---|---|
| **한눈에 파악** | 상단에 TL;DR / 핵심 메시지 |
| **계층 구조** | H1 (제목) → H2 (큰 섹션) → H3 (소제목) |
| **시각 도움** | 테이블 / 코드 블록 / 이모지 / 아이콘 |
| **간결함** | 한 섹션 = 한 메시지 |
| **링크** | 다른 정본 문서로 이동 가능 |
| **변경 이력** | 하단에 날짜별 변경 기록 |

### 길이 가이드

| 문서 종류 | 권장 길이 |
|---|---|
| 핵심 원칙 (CORE_VISION 등) | 200-300줄 |
| 실행 지침 (EXECUTION) | 300-500줄 |
| 전략 (EXPANSION 등) | 200-400줄 |
| 워크플로우 (LLC_SETUP 등) | 300-400줄 |
| 인덱스 (README) | 100-200줄 |

**500줄 초과 시:** 주제별로 분할 검토

## 🚨 중복 위험 신호

다음이 발견되면 즉시 정리:

```
🚨 같은 원칙이 3개 이상 문서에 나옴
🚨 같은 표·코드 블록이 여러 곳에 복사됨
🚨 한 문서 안에서 같은 메시지 반복
🚨 어느 문서가 정본인지 불명확
🚨 README가 실제 문서와 동기화 안 됨
🚨 변경 이력이 1년 이상 업데이트 안 됨 (이 문서 더 이상 사용 안 됨 시그널)
```

## 📋 자동 위생 체크리스트

새 문서 추가·기존 문서 큰 변경 후 매번 점검:

```
[추가된 내용]
□ 새 인사이트가 정본 문서에 들어갔는가?
□ 다른 문서에 같은 내용이 이미 있는가?
□ 있다면 정본만 남기고 다른 곳은 링크로 변경했는가?

[시각 품질]
□ 한 섹션이 한 메시지에 집중되는가?
□ 테이블·코드 블록 등 시각 요소 사용했는가?
□ 길이가 권장 범위인가?
□ 한·영 혼용 시 일관된 표기인가?

[연결성]
□ 관련 문서 링크가 정확한가?
□ README.md에 추가/수정 반영했는가?
□ CLAUDE.md 인사이트 트래커 업데이트했는가?

[변경 이력]
□ 하단 변경 이력에 날짜·요약 추가했는가?
□ "마지막 업데이트" 날짜 갱신했는가?
```

## 🔄 분기별 종합 정리 (매 분기 첫 주)

분기 리셋 시 폴 김 목사가 점검:

```
□ 모든 문서 읽기 (또는 Claude Code에 요약 요청)
□ 정본 매핑 표 (이 문서 위쪽) 최신화
□ 사용 안 되는 문서는 archive로 이동
□ 새로 발견된 위험 신호 처리
□ 길이 500줄 초과 문서 분할 검토
□ 시각 품질 일관성 점검
```

**Archive 위치:** `docs/archive/YYYY-Q#/`

## 💡 인사이트 추가 시 표준 프로세스

폴 김 목사가 새 인사이트를 공유할 때 Claude Code 자동 실행:

```
1. 어디에 들어갈지 결정
   - 기존 정본 문서에 추가 / 새 문서 생성
   
2. 추가하면서 기존 중복 자동 검색
   - grep으로 유사 키워드 찾기
   - 발견 시 통합 또는 링크 처리

3. 시각 품질 자동 적용
   - 테이블·아이콘·코드 블록 일관 사용
   - 계층 구조 유지

4. 연결 문서 자동 갱신
   - README.md 업데이트
   - CLAUDE.md 인사이트 트래커
   - 관련 문서 cross-link

5. 변경 이력 자동 기록
   - 정본 문서 하단
   - 날짜·요약·출처

6. Git 커밋·푸시
   - 의미 있는 commit message
   - 한 번에 일관된 변경
```

## 🌟 영혼 — 왜 깔끔해야 하는가

> *"내가 사용자라면, 어수선한 정보 더미를 받고 싶을까?  
> 아니면 명료하고 아름답게 정리된 가이드를 받고 싶을까?"*

황금률(마7:12)이 문서에도 적용된다.  
사역자·미래의 자신·새 세션의 Claude 모두 사용자다.  
그들에게 **한눈에 파악되는 깔끔한 문서**를 제공하는 것이 환대다.
