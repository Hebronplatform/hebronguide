## 핵심 데이터 (Yelp × Harvard 연구)

> **"Yelp 별점 1점 상승 = 음식점 매출 5.4% 증가"**  
> — Harvard Business School, 미국 워싱턴주 식당 6만 개 × 7년 데이터 분석

**HebronGuide에 주는 의미:**
- 별점 + 리뷰 수는 가장 강력한 신뢰 지표 → 모든 카드에 표시 필수
- 검증된 데이터 기반 소개 = 사용자 신뢰 = 재방문율 증가

## 1. 소비자 관점 설명 원칙 (가장 중요)

### ❌ 하지 말 것 — 사실 나열
```
"✅ 검증됨 | 한인 BBQ. 3000 184th St SW | ☎ (425) 490-6328 | 월-목 11:30am-10pm"
```

### ✅ 해야 할 것 — 소비자 가치 먼저
```
"고기 무한리필 + 직원이 직접 구워줌. 무제한 밑반찬. 알더우드몰 주차 무료.
📍 3000 184th St SW | 📞 (425) 490-6328 | 월-목 11:30am-10pm"
```

### 소비자 관점 작성 공식

```
[이럴 때 가세요] + [이게 특별한 이유] + [실용 정보]
```

**예시:**
| 상황 | 소개 방식 |
|---|---|
| BBQ | "삼겹살·갈비 먹고 싶을 때. 직원이 직접 구워줌. 무제한 리필" |
| 순두부 | "해장할 때 1순위. 진한 국물. 양 많아서 배부름" |
| 카페 | "한국 카페 분위기 그리울 때. 빙수·크로플·달달한 음료" |
| 치킨 | "치맥 하고 싶을 때. 바삭한 한국식 양념치킨. 배달도 됨" |
| 마트 | "한국 재료 구하러. H-Mart 쇼핑 후 함께 방문 가능" |

## 2. 카테고리 퀵 필터 운영 원칙

### 기준 카테고리 (모든 도시 공통)

| 필터 | 이모지 | 해당 음식 | 태그 키워드 |
|---|---|---|---|
| 전체 | 🍽️ | 모든 음식 | - |
| BBQ·갈비 | 🥩 | 삼겹살·갈비·KBBQ | "BBQ" |
| 치킨 | 🍗 | 한국식 후라이드·양념·간장치킨 | "치킨" |
| 국물·찌개 | 🍲 | 순두부·갈비탕·설렁탕·해장국 | "국물" |
| 카페·디저트 | ☕ | 한국식 카페·빙수·크로플·케이크 | "카페" |
| 분식·기타 | 🍜 | 라면·떡볶이·김밥·만두 | "분식" |

### 카테고리 태그 부여 규칙
- 모든 음식점 tags 배열에 카테고리 키워드 반드시 포함
- 예: `tags: ["BBQ", "갈비", "린우드", "검증됨"]`
- 복수 카테고리 가능: `tags: ["BBQ", "치킨", "링우드"]`

## 3. 별점·리뷰 수 표시 원칙

### 표시 형식
```
name: "Baekjeong Korean BBQ ★ Yelp 3.9"
```
또는 Top5Banner에서:
```typescript
{ rating: 4.2, ratingCount: "800+" }
```

### 별점 소스 우선순위
1. **Yelp** — 한인 업소 리뷰 가장 신뢰도 높음
2. **Google Maps** — 전반적 리뷰 수 많음
3. 검증 안 된 경우 → 별점 표시 생략 (허위 정보 금지)

### 검증 표시 체계
| 표시 | 의미 |
|---|---|
| `★ Yelp 4.2 (800+)` | Yelp 검증된 평점 |
| `✅ 검증됨` | 직접 확인 완료 |
| `🔗 yelp.com/biz/...` | Yelp 페이지 직링크 |
| (표시 없음) | 미검증 — 확인 필요 |

## 4. 링크 체계 원칙

### 모든 음식점 카드 필수 링크
```
📍 Google Maps 버튼 (주소 있을 때)
📞 전화 버튼 (전화번호 있을 때)
🌐 공식 웹사이트 또는 Yelp 링크
```

### Yelp 링크 형식
- 직접 페이지: `🔗 yelp.com/biz/restaurant-name-city`
- 검색 링크: `🔗 yelp.com/search?find_desc=음식점명+도시`

### Yelp가 없을 때 대안
- Google Maps 리뷰 링크: `maps.google.com/?q=음식점명+도시`
- 네이버 지도 (한인 업소): `map.naver.com/?query=...`
- 공식 홈페이지 우선

## 5. 콘텐츠 신뢰성 원칙 (Yelp에서 배운 것)

### Yelp의 핵심 차별화: 리뷰 조작 불가
HebronGuide 적용:
- 업소가 직접 수정 요청해도 검증 없이 변경 금지
- 검증 날짜 기록 필수 (`verified_at: YYYY-MM-DD`)
- 폐업·이전 업소는 즉시 "정보 확인 중"으로 변경

### 정보 투명성
- 광고성 콘텐츠는 명확히 표시 (향후 스폰서십 시)
- "이 업소의 정보를 제공해 주신 분께 감사합니다" 등 출처 명시
