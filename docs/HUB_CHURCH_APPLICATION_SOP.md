## 🔄 4단계 운영 흐름

### Step 1 — 신청 접수 확인 (24시간 내)

신청 이메일 도착 시:

```
1. 운영자 (폴 김 목사 또는 위임자) 확인
2. 24시간 내 자동·수동 응답 발송
3. 트래킹 시트에 신청 등록 (도시·교회명·접수일·상태)
```

**누락 정보 처리**: 자가 진단 양식 미작성·일부 누락 → "추가 정보 요청" 템플릿 발송.

### Step 2 — 4-Tier 분류 (1-3일)

운영자가 자가 진단 + 자체 조사로 Tier 결정:

```
✅ 명확 Tier 1-3 → Step 3 (신앙 검증) 진행
🔍 Tier 4 (독립·무소속) → 폴 김 목사 1차 검토 의뢰
🚫 이단 의심 → 즉시 거부 응답 (정중하게)
```

**조사 자료**:
- 교회 공식 웹사이트 (교단·신앙고백 명시 여부)
- 한인 교회 디렉토리 (해당 도시 한인 미디어 검색)
- 폴 김 목사 인적 네트워크 조회 (필요시)

### Step 3 — 신앙·환대 동행 검증 (1-2주)

자가 진단 양식 답변 검토 — **반드시 내부 운영팀만 검토. 앱·외부 노출 ❌** (마 10:16 지혜·순결):

```
[복음 신앙 고백 — 동의 필수, 내부 검토용]
□ 사도신경 동의
□ 성경 무오성 동의
□ 그리스도의 십자가 대속·부활 신앙

[환대 동행 약속 — 내부 검토용]
□ 새 이민자 환대 동행자 역할 동의
□ 사역자·전문가 추천 협력 의사
□ 분기별 영혼 점검 자가 진단 참여
```

**1개라도 ❌** → 정중한 보류 응답 + 추가 대화 제안 (필수 시 1:1 줌 미팅).

> ⚠️ **공개 라벨 금지**: 위 자가 진단 결과는 운영팀이 매칭·등재 결정에만 활용.
> 교회를 "신앙 ✅", "환대 동행 ✅" 등으로 **공개 라벨링 ❌** — 마 10:16 어긋남,
> 양쪽(교회·신청자) 모두 무안. 라벨 없이 desc 본문 표현으로 자연스럽게.

### Step 4 — 등록 + 환영 (즉시)

승인 결정 시:

```
1. 승인 응답 발송 (환영 + 다음 단계 안내)
2. 트래킹 시트 → 상태 "승인" + 등록일
3. HebronGuide 앱 데이터에 허브교회 등록 (도시·교회명·연락처·Tier)
4. 다음 분기 영혼 점검 일정 발송
```

**Phase 2 자동화 예정**: Supabase에 `hub_churches` 테이블 → 앱이 실시간 조회.

## 🛡️ Hard Rules — 절대 금지

```
❌ 자가 진단 미작성 자동 등록
❌ 검증 없이 앱에 노출
❌ 이단 등재 (신천지·통일교·JMS·하나님의교회·여호와의증인·몰몬교)
❌ 한국 우월주의·민족주의 강조 교회 등재
❌ 정치적 좌·우 편향 콘텐츠 강조 교회 등재
❌ 무슬림·다민족 직접 표적 마케팅 (HebronGuide는 한인 디아스포라 일차 사명)
```

## 💡 영혼 점검 (등재 결정 전 5질문)

```
□ Q0 — 내가 새 이민자라면, 이 교회에 도착했을 때 환영받을까?
□ Q3 — 이 교회는 새 이민자를 수단(교인 수)으로 보지 않는가?
□ Q4 — 이 교회의 신앙 고백이 정직하게 명시되어 있는가?
□ Q5 — 이 교회에 도착하는 이민자가 그리스도의 환대를 자연스럽게 경험할까?
□ Q9 — 이 교회의 사역 전체가 송영(Doxology)을 향하는가?
```

5개 모두 ✅일 때만 등재.
