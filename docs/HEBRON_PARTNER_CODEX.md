## 서문 · 모든 것의 기초

> *"그런즉 너희가 먹든지 마시든지 무엇을 하든지 다 하나님의 영광을 위하여 하라"*  
> — 고린도전서 10:31

우리는 **왕의 자녀**입니다.

먹는 것도, 마시는 것도, 일하는 것도, 돈 버는 것도, 사람을 연결하는 것도 —  
**무엇을 하든지** 하나님의 영광을 위해 합니다.

이것이 HebronGuide의 가장 깊은 뿌리입니다.

```
라이드 기사가 운전할 때          → 하나님의 영광을 위해
호스트가 방을 내어줄 때          → 하나님의 영광을 위해
튜터가 아이를 가르칠 때          → 하나님의 영광을 위해
가이드가 도시를 안내할 때         → 하나님의 영광을 위해
매칭으로 두 사람이 만날 때        → 하나님의 영광을 위해
목사님이 새가족을 환영할 때       → 하나님의 영광을 위해
수익을 나눌 때                  → 하나님의 영광을 위해
```

이 하나의 질문이 모든 결정의 기준입니다:  
**"이것이 하나님께 영광이 되는가?"**

### 왕의 자녀가 섬긴다는 것

왕의 자녀는 두렵거나 궁핍해서 섬기지 않습니다.  
**풍성함에서** 섬깁니다.

```
종의 섬김:  "해야 하니까" → 강요, 두려움, 소진
왕자의 섬김: "하고 싶으니까" → 기쁨, 자유, 지속
```

우리가 하는 모든 서비스는  
**왕의 자녀가 왕의 나라를 위해 기쁨으로 하는 섬김**입니다.  
구걸하지 않습니다. 사과하지 않습니다.  
당당하게, 따뜻하게, 진심으로 섬깁니다.

## 2장 · 성경적 기초 — 6개의 기둥

### 기둥 1: 환대 (마태복음 25:35)
```
"내가 나그네 되었을 때 너희가 영접하였다"
```
환대는 선택이 아닙니다. **예수님이 나그네의 모습으로 우리 앞에 서 계십니다.**  
HebronGuide의 모든 서비스(라이드·스테이·튜터·관광·커뮤니티·매칭)는  
이 한 구절의 실천입니다.

### 기둥 2: 황금률 (마태복음 7:12)
```
"무엇이든지 남에게 대접을 받고자 하는 대로 너희도 남을 대접하라"
```
**내가 낯선 도시에 혼자 도착했을 때 받고 싶은 것을 드리는 것.**  
이것이 모든 의사결정의 첫 번째 질문입니다.

### 기둥 3: 지혜와 순결 (마태복음 10:16)
```
"뱀처럼 지혜롭고 비둘기처럼 순결하라"
```
- 지혜: 현실적이고 효과적으로 섬깁니다. 좋은 의도만으로 부족합니다.  
- 순결: 목적이 변질되지 않습니다. 돈도, 성장도, 명예도 목적이 아닙니다.

**이 두 가지가 없으면 선한 의도가 나쁜 결과를 냅니다.**

### 기둥 4: 제자 삼는 사명 (마태복음 28:19)
```
"모든 족속으로 제자를 삼으라"
```
HebronGuide는 한인 이민자를 섬기지만 — 궁극적으로는 제자를 만드는 일을 합니다.  
서비스가 목적이 아닙니다. **제자가 목적이고, 서비스는 문입니다.**

### 기둥 5: 관계를 통한 확산 (요한복음 1:41)
```
"안드레가 먼저 자기 형제 시몬을 찾아 데리고 예수께로 오니"
```
강제 전도가 아닙니다. **내가 만난 것을 사람에게 소개하는 것.**  
라이드 기사가 교회를 소개합니다. 호스트가 삶으로 보여줍니다.  
관광가이드가 이 도시의 이야기를 나눕니다.  
모든 연결이 자연스럽게 교회 공동체로 이어집니다.

### 기둥 6: 코이노니아 (사도행전 2:44-45)
```
"믿는 사람이 다 함께 있어 모든 물건을 서로 통용하고"
```
HebronGuide 파트너십은 **이익을 나누는 비즈니스**이면서  
동시에 **코이노니아의 디지털 구현**입니다.  
라이드 수입, 스테이 수입, 튜터 수입 — 이 나눔이 성경적 공동체의 표현입니다.

## 3장 · 철학적 기초 — 우리가 하지 않는 것

### 우리는 하지 않습니다

```
❌ 직접 전도 — 서비스 중 복음을 강요하지 않습니다
❌ 교회 홍보 — 특정 교회를 밀어 넣지 않습니다
❌ 수단화 — 사람을 교회 성장의 도구로 쓰지 않습니다
❌ 불투명 — 파트너 수입과 사용처를 숨기지 않습니다
❌ 경쟁 — 허브교회끼리 경쟁하지 않습니다
❌ 이단화 — 신학적으로 타협하지 않습니다
```

### 왜 우리는 "교회 검증"을 광고하지 않는가

사용자가 "교회 검증"을 보는 순간 — 경계합니다.  
종교적 목적이 있다고 느낍니다.

**진짜 신뢰는 광고하지 않아도 느껴집니다.**  
따뜻한 드라이버, 따뜻한 호스트 가정, 진심으로 가르치는 튜터 —  
이것 자체가 신뢰이고, 이것이 교회로 이어집니다.

```
광고된 신뢰:  "교회 검증 드라이버" → 경계
느껴지는 신뢰: "정말 따뜻했어요" → 교회 가고 싶다
```

### 왜 커미션을 노출하지 않는가

비즈니스 모델을 앞에 세우는 순간 — 섬김이 거래가 됩니다.  
거래는 관계를 낳지 못합니다.

**수익은 결과이지 목적이 아닙니다.**  
섬김이 먼저이고, 지속가능한 섬김을 위해 수익을 나눕니다.

## 5장 · 파트너 관계 — 함께 섬기고 함께 나눕니다

### 파트너의 종류

```
Level 1: 허브교회 (Hub Church)
  → 새가족 연결, 도시 커버리지
  → 연결된 사람들의 교회 정착 책임

Level 2: 서비스 파트너 (Service Partner)
  → 라이드·스테이·튜터·관광가이드·커뮤니티·매칭
  → 직접 서비스 제공 + 자연스러운 교회 연결

Level 3: 목회자·선교사 네트워크
  → 도시 간 연결, 전략 파트너십
  → 분기별 네트워크 미팅 참여
```

### 수익 나눔 원칙

```
원칙 1: 투명성
  모든 수입과 배분은 파트너에게 투명하게 공개됩니다

원칙 2: 공정성
  서비스 제공자 → 적정 수입 보장
  HebronGuide → 플랫폼 운영·개발
  허브교회 → 연결 역할에 대한 기여 인정

원칙 3: 사명 우선
  수익이 많아져도 사명이 변질되지 않습니다
  수익이 적어도 섬김을 멈추지 않습니다

원칙 4: 나눔
  순이익의 일부는 재외동포 사역 지원에 사용됩니다
```

### 파트너가 받는 것

```
허브교회:
  ✓ 새가족 사전 알림 (도착 전 연락)
  ✓ 전국·전 세계 목회자 네트워크
  ✓ HebronGuide 앱 허브교회 표시
  ✓ 서비스 수입 중 연결 기여분 공유
  ✓ 도시 인구통계·정착 트렌드 데이터

서비스 파트너:
  ✓ 고객 매칭 플랫폼
  ✓ 결제·일정 관리 시스템 (구축 예정)
  ✓ 한인 커뮤니티 신뢰 브랜드 활용
  ✓ 서비스 수입의 적정 비율 (투명하게 공개)
```

## 6장 · 흔들리지 않기 위해

### 예상되는 유혹과 함정

#### 함정 1: 수익 극대화 유혹
```
"더 많이 받으면 더 많이 섬길 수 있다"
→ 거짓입니다. 더 많이 받으면 섬김의 동기가 바뀝니다.
→ 대응: 수익 목표를 고정하고 그 이상은 사명에 씁니다
```

#### 함정 2: 성장 지표 집착
```
"사용자 수, 다운로드 수가 우리의 성공이다"
→ 아닙니다. 교회에 연결된 사람이 우리의 성공입니다.
→ 대응: 핵심 지표는 항상 '교회 연결된 사람 수'
```

#### 함정 3: 교회 경쟁
```
"우리 교회가 더 많은 새가족을 받아야 한다"
→ 허브교회끼리 경쟁이 시작되면 네트워크가 죽습니다.
→ 대응: 도시 지역 분담, 파트너 교회 간 경쟁 금지
```

#### 함정 4: 종교 기업 변질
```
"HebronGuide가 너무 커지면 교회보다 플랫폼이 먼저가 된다"
→ 가장 위험한 함정입니다.
→ 대응: 분기별 자가점검 — "우리가 교회를 섬기는가, 교회가 우리를 섬기는가?"
```

#### 함정 5: 교단·신학 타협
```
"더 많은 교회를 포함하기 위해 기준을 낮추자"
→ 기준이 낮아지면 신뢰가 낮아집니다.
→ 대응: 허브교회 6가지 기준은 타협하지 않습니다
```

### 파트너 자가점검 질문 (분기별)

```
□ 나는 여전히 섬김을 위해 이 일을 하는가?
□ 수입이 나의 동기를 바꾸고 있지 않은가?
□ 내가 연결한 사람들이 실제로 교회에 정착했는가?
□ 나는 HebronGuide를 비즈니스로 보는가, 사역으로 보는가?
□ 파트너들과의 관계가 거래적이 되지 않았는가?
□ 이 일이 기쁨이 있는가, 의무가 되었는가?
```

## 7장 · 파트너 서약

이 코덱스를 읽고 동의하는 파트너는 다음을 서약합니다:

```
나는 HebronGuide 파트너로서:

1. HebronGuide의 사명(디아스포라 환대·교회 연결)을
   나의 사역과 일치시킴을 확인합니다.

2. 서비스를 통해 만나는 모든 사람을
   교회와 공동체로 자연스럽게 연결하기 위해 힘씁니다.

3. 수익보다 관계를, 효율보다 진정성을 앞세웁니다.
```

## 🤝 Hebron 정신의 핵심 — 섬김과 관계

> *"인자가 온 것은 섬김을 받으려 함이 아니라*  
> *도리어 섬기려 하고 자기 목숨을 많은 사람의 대속물로 주려 함이니라"*  
> — 마가복음 10:45

### 로봇이 주목받는 세상에서

AI가 그림을 그리고, 로봇이 배달하고, 알고리즘이 추천하는 시대.

그러나 **사람은 역시 사람 사이의 관계가 중요합니다.**

```
로봇이 대체할 수 없는 것:
  → 새벽에 도착한 이민자를 안아주는 손
  → 처음 낯선 도시에서 같이 밥 먹어주는 사람
  → 아이가 학교 적응 못 할 때 같이 앉아 들어주는 이웃
  → 힘든 날 옆에 있어주는 커뮤니티

Hebron이 하는 일이 바로 이것입니다.
```

### 예수님의 섬김 모델

예수님은 하나님이시지만 사람으로 오셨습니다.  
섬김을 **받으러** 오신 게 아니라 섬기러 오셨습니다.

```
Hebron 파트너의 정체성:
  예수님처럼 → 섬기러 온 사람
  왕의 자녀로서 → 두렵지 않게, 기쁨으로
  이웃에게 → 관계로, 사람으로
```

### 섬김 + 관계 = Hebron 정신

```
섬김 (Service):
  • 도구가 아니라 마음으로
  • 효율이 아니라 관계로
  • 거래가 아니라 선물로

관계 (Relationship):
  • 앱 연결로 시작하지만
  • 사람과 사람으로 이어지고
  • 공동체로 자란다

이 둘이 만날 때 → 헤브론
```
