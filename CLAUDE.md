## ⚡ 성능 — 병렬 처리 원칙

**모든 I/O 작업은 기본적으로 병렬 처리한다.**

### 코드 작성 시
- URL 점검, API 호출, 파일 읽기 등 독립적인 작업은 반드시 병렬로 처리
- Python: `concurrent.futures.ThreadPoolExecutor` 사용 (기본 workers=20)
- JavaScript: `Promise.all()` 또는 `Promise.allSettled()` 사용
- 순차 처리(for loop + sleep)는 의존성이 있는 경우에만 허용

```python
# ✅ 항상 이렇게
from concurrent.futures import ThreadPoolExecutor, as_completed
with ThreadPoolExecutor(max_workers=20) as executor:
    futures = {executor.submit(check_url, url): url for url in urls}
    for future in as_completed(futures):
        result = future.result()

# ❌ 절대 이렇게 하지 않음
for url in urls:
    result = check_url(url)
    time.sleep(0.3)
```

- 달라스, LA, 뉴욕 등 여러 도시 데이터를 동시에 수집할 때 병렬 처리 필수

- 순서 의존성이 없으면 절대 순차 실행하지 않음

## 🧭 데이터 품질 3원칙 (Codex 기반)

HebronGuide가 교차로·AI·일반 검색과 차별화되는 **"신뢰 인프라"**의 핵심.
모든 콘텐츠는 다음 3단계 검증을 거쳐야 한다.

### 검증 표시 체계

| 표시 | 의미 |
|------|------|
| `✅ 검증됨` (녹색 체크) | 공식 사이트·직접 확인 완료 |
| `🔗 웹사이트` (링크) | 사용자가 직접 최신 정보 확인 가능 |
| (표시 없음 / `정보 확인 중`) | 아직 미검증 → **공개 보류** |

### 5단계 운영 절차

```
1. AI가 공식 웹사이트 검색 → 1차 수집
2. 전화번호·주소 개별 대조 확인 (직접 통화 또는 공식 페이지)
3. ✅ 검증 완료 후 Admin 패널에서 입력
4. 날짜 기록 → "마지막 확인: YYYY-MM-DD"
5. annual_update.py → 연 1회 자동 점검 (매년 1월)
```

## ✝️ Hebron 정신 — 히든 미션 (Claude Code 필독)

> **이 섹션은 HebronGuide의 모든 기능·콘텐츠·UX 설계의 근본 철학이다.**

### 대표 구호의 3중 의미

**"도시를 알고, 사람을 찾다"**

```
1층 — 일반인이 인식하는 것:
  실용적 도시 정착 가이드 (맛집·정착·취업·관광)

2층 — 한인 커뮤니티가 느끼는 것:
  같은 뿌리를 가진 사람들의 연결과 소속감

3층 — 히든 미션 (드러내지 않되 스며들게):
  사람과 하나님의 연결 — 그리스도 제자 공동체로의 초대
```

### 성경적 근거

> **"내가 나그네 되었을 때 너희가 영접하였다"** (마태복음 25:35)

HebronGuide 자체가 이 말씀의 디지털 구현이다.  
이민자·이주자·방문자 = 현대의 나그네.  
HebronGuide = 그들을 영접하는 디지털 환대(hospitality) 사역.

### Hebron 지명의 의미

헤브론(Hebron) = 히브리어 "연합·교제·동반자" (חֶבְרוֹן)  
아브라함이 거류민으로 살며 하나님과 동행한 땅.  
HebronGuide = 이민자들이 새 땅에서 하나님과 동행하도록 돕는 플랫폼.

### 실천 원칙 (Claude Code 작업 시 적용)

### ✝️ 교회 사역자 영업 권한 정책 (Pastor Authorization Policy)

**원칙**: 교회 집사·사역자·자원봉사자가 HebronGuide를 대표하여 영업 활동(업소 등재 제안, 웹사이트 제작 영업, 파트너십 협상)을 할 경우, **반드시 담임목사의 서면 동의(서명)**를 받아야 한다.

#### 필수 요건

| 요건 | 내용 |
|------|------|
| **담임목사 서명** | 영업 권한 부여 공문에 담임목사 친필 서명 |
| **유효 기간** | 서명일로부터 **1년** |
| **연간 갱신 (Renew)** | 매년 갱신 — 갱신 없으면 자동 권한 소멸 |
| **갱신 시기** | 만료 30일 전 교회에 갱신 요청 |
| **적용 범위** | HebronGuide 명의로 하는 모든 상업적 접촉 |

#### 이유 (Why This Matters)

```
① 투명성 — 교회 이름으로 상업 활동 = 목사 인지·동의 필수
② 책임 — 문제 발생 시 명확한 책임 소재
③ 남용 방지 — 무단으로 교회 신뢰를 이용한 영업 차단
④ 목양적 보호 — 집사·사역자가 개인 이익으로 변질되지 않도록
⑤ Hebron 정신 — "교회가 신뢰 앵커" = 목사가 그 신뢰를 보증
```

#### 담임목사 서명 공문 양식 (필수 항목)

```
[HebronGuide 사역자 영업 권한 부여서]

교회명: _______________
담임목사: _____________
권한 부여 대상자: _______  직분: ________
권한 내용: HebronGuide 관련 업소 등재 제안, 
           웹사이트 제작 영업, 파트너십 협의
유효 기간: ____년 __월 __일 ~ ____년 __월 __일 (1년)
갱신 조건: 만료 전 담임목사 재서명 필요

담임목사 서명: ____________  날짜: __________
```

#### 연간 갱신 절차

```
1월: HebronGuide 본부 → 각 교회 담임목사에게 갱신 안내
2월: 교회 → 갱신 서명 공문 제출
3월: HebronGuide → 갱신 확인 + 다음 1년 권한 활성화

갱신 미제출 시:
  → 자동 권한 소멸
  → 영업 활동 즉시 중단
  → 갱신 후 재활성화
```

#### 수수료 지급 원칙

- 수수료는 **사역팀 개인이 아닌 교회 계좌로** 지급 (기본)
- 교회가 사역자에게 재분배 방식은 교회 내부 결정
- 이유: 개인 이익 추구 동기 제거, 교회 사역 재원으로 활용
- 예외: 담임목사가 개인 지급 허용 시 서면 명시

## 📐 HebronGuide 운영 5대 원칙

### 원칙 1. 정착자의 눈 (Settler's Eye)
> "관광객이 아닌, 그 도시에 살아본 한인이 쓴다"
- 모든 정보는 해당 도시 거주 한인이 직접 경험·검증한 것만 등재
- 출처: Time Out "로컬 전문가" + Lonely Planet "직접 방문" 원칙 결합

### 원칙 2. 도착 첫날부터 (Day-1 Ready)
> "공항에서 나온 그 순간부터 쓸 수 있어야 한다"
- 이민 생애주기별 정보 구조화:
  - 0~3개월 (정착기): 비자·면허·은행·SIM·한인 병원·교회
  - 3~12개월 (적응기): 학교·직장·커뮤니티 네트워크
  - 1년+ (정주기): 부동산·시민권·자녀 교육

### 원칙 3. 교회가 신뢰 앵커 (Church as Trust Anchor)
> "광고로 살 수 없는 신뢰를 교회 네트워크가 만든다"
- 파트너 교회가 검증한 업소는 별도 배지 표시
- 출처: Nextdoor "실명 인증" 원칙을 한인 커뮤니티 방식으로 구현

### 원칙 4. 투명성이 권위다 (Transparency = Authority)
> "누가, 언제, 어떻게 확인했는지를 항상 밝힌다"
- 모든 정보에 명시: 마지막 확인 날짜 / 확인 방법 / 이해충돌 여부
- 출처: Michelin "익명 검증" + Yelp "신뢰 시스템 공개" 원칙

### 원칙 5. 현지인과 함께 (With Locals, Not Just For Koreans)
> "한인 가이드가 영어권 현지인도 찾는 플랫폼이 된다"
- 한인 문화를 현지인에게 소개하는 영어 콘텐츠 병행
- 이민자와 현지인이 함께 사는 세상을 만드는 인프라

### 수익 모델 로드맵
| 단계 | 수익원 | 참고 플랫폼 |
|------|--------|-----------|
| 초기 (0~1년) | 교회 파트너십, 무료 시작 | Meetup 신뢰 우선 전략 |
| 성장기 (1~2년) | 한인 비즈니스 프리미엄 리스팅 | HeyKorean 플래티넘 모델 |
| 확장기 (2~3년) | 스폰서 가이드 (법무사·부동산·보험) | Expatica 디렉터리 모델 |
| 성숙기 (3년+) | B2B 기업 파트너십 + 웹사이트 제작 | InterNations 기업 파트너 |
